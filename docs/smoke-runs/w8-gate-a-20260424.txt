  - emitEvent param renamed: payload → details (matches jsonb column).
  - to_status is synthesized from the event enum via a translation map
    DB_EVENT_NAMES : TransitionEventType → string. Writes use a
    partb_* prefix for namespace isolation from Phase 3.5 state values.
    event_type AND to_status both get the same translated string

    columns to varchar(64) and collapse the map to a simple prefix,
    but that's a schema change so it's out-of-scope for this fix.
  - try/catch .warn wrapper removed; emitEvent now console.error's
    the supabase error and throws. Matches Rule 38 (loud shadow-mode
    observability). Fire-and-forget dispatcher's outer .catch still
    prevents the orchestrator from taking down the request handler.
