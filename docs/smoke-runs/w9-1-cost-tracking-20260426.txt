  shadow_runs column            $0.563500
  delta                         $0.367969

The delta is expected behavior, not a regression:
  - shadow_runs.part_b_cost_usd persists totalCost(ctx.costAccumulator),
    which `+=` accumulates across every agent invocation in the run
    (including all 2 revise iterations × 4 agents = ~8 extra calls beyond
    the final-cycle snapshot).
  - The per-agent `cost_usd` fields in the JSONB blob reflect each agent's
    most-recent successful call only (mutation-after-parse pattern). They
    are not cumulative.
  - Total invocations = 11; per-agent JSONB shows 4 final-cycle calls; the
    other 7 calls' cost is captured in the column but not in the snapshot.

The two surfaces have different semantics by design:
  - part_b_cost_usd column   → cumulative spend for the run (auditable)
  - per-agent JSONB cost_usd → last successful agent output's cost (debug
                                aid, mirrors how `metadata.retry_count`
                                is mutated post-parse)
